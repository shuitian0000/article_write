## 1. 选题方法论

### 选题交集模型

一个好选题 = 一个真问题 + 你的私人入口 + 可挖的深度

光是"真问题"，可能太私密，读者进不去。光是"私人入口"，可能流于自传。光是"可挖深度"，可能太抽象，没人爱看。三者结合才能找到最精准的引爆点。

### TSD质检法

动笔前每个选题必须通过：

- **T (True question 真问题)** 这是不是一个你自己真的困惑过、想过多遍的问题？还是为了写文章硬凑的题目？真问题能让你写的时候也有"对哦，到底是为什么"的感觉。
- **S (Substance 实质内容)** 有没有实质的东西可讲？——一段个人经历、一个具体场景、一本书/一个观点、一组可观察的现象？空对空谈"成长""意义"是死路。
- **D (Depth 深度可挖)** 能不能往深挖至少两层？第一层是表面解释，第二层是机制/结构，第三层是哲学/存在层面的洞察。如果只能停在第一层，选题太浅。

S级选题三项兼备，及格至少两项，只占一项要重新审视，零项坚决放弃。

### 三不写（反向否决）

- **不是自己真想过的题不写**：为了蹭热点硬凑的选题，写出来一定是装的。读者一眼能看出来。
- **只有"道理"没有"经历"不写**：纯道理输出等于鸡汤。必须有具体场景、具体困惑、具体人物支撑。
- **一句话能说清楚却硬要拉成文不写**：如果一个想法用一条朋友圈就能讲清楚，硬拉成3000字一定是注水。深度文章不是字数堆砌，是思考密度的体现。

### 选题来源

**自我观察**：自己最近困扰的事、反复想的问题、突然困惑的瞬间——这是最珍贵的来源，因为只有你自己能写。

**读书札记**：读到的书里让你愣一下的段落、突然照亮自己经历的概念、让你反复回想的观点——书的锚点+你的经历，天然适合札记式读书型。

**日常现象**：身边的人都在做的事、突然出现的新现象、让你"诶？"一下的小细节——适合现象追问型。

**公共议题**：当代人精神困境、现代病、代际差异、时代情绪——需要私人入口，不能空谈。

**读者社群**：群友的困惑、留言区反复出现的问题、私信里被问到的难题——监测"怎么破""怎么办""是不是只有我"等关键词。

### 角色代入法（发散思考）

审视选题时代入不同角色：
- "一个被困扰过的普通人"：这个困惑真的成立吗？我自己真的被困扰过吗？
- "一个聪明的读者"：这个选题能让我读完有"对，我也是"或"原来还可以这样想"的瞬间吗？
- "一个反对者"：这个观点的反方会怎么说？我能把反方讲透吗？

## 2. 选题分类与适配原型

### 按母题分类

**现代人精神困境**（注意力碎片、信息过载、职业倦怠、意义感缺失、拖延、自我怀疑）
- 适配原型：困境切入型 / 现象追问型
- 写法核心：困境具象化 → 机制剖析 → 结构性洞察
- 慎防：变成抱怨文或鸡汤文

**读书札记**（一本书/一个思想家的核心观点结合个人体验）
- 适配原型：札记式读书型
- 写法核心：个人困惑铺垫 → 书的核心观点 → 观点照亮经历 → 自己的修正或反对
- 慎防：变成书评或读后感作业

**日常生活的现象学**（从一个生活事件引出深度思考）
- 适配原型：困境切入型 / 现象追问型
- 写法核心：事件描述 → 第一层解读 → 追问 → 更深解读 → 哲学升维
- 慎防：停在"这事儿挺有意思"的观察层

**心理/哲学概念解读**（把一个概念讲透并落到个人成长）
- 适配原型：概念解剖型
- 写法核心：生活场景引入概念 → 准确表述 → 生活化举例 → 反例和边界 → 个人体悟
- 慎防：变成科普文或教科书

**方法论分享**（自己长期实践的方法/原则）
- 适配原型：方法论型（自我实践）
- 写法核心：谦逊铺垫 → 具体做法 → 真实改变 → 学习成本和失败点 → 适用边界
- 慎防：夸大效果或隐瞒失败点

### 按内容类型

**深度反思**：对一个问题反复思考后写出
**读书札记**：以书为锚点的思考
**现象解读**：从一个现象切入深挖
**概念剖析**：讲透一个心理学/哲学概念
**方法论分享**：自我实践的方法

### 按写作原型

**困境切入型**
从一个具体的个人困境切入，层层剖析。最能建立信任感和叙事张力。
- 失眠那个晚上 → 从具体的失眠场景，层层剖析到现代人的精神困境
- 突然不想起床的某个早晨 → 从一个具体状态，剖析倦怠的机制
写法核心：困境具象化 → 表面解释 → 深层机制 → 结构性洞察 → 出路或承认无解
小标题：禁止。靠氛围和心流推进。

**札记式读书型**
以一本书/一个思想家为锚点，结合个人体验展开。不是书评，是"这本书帮我看见了自己"。
- 读《倦怠社会》想到自己的996
- 读《被讨厌的勇气》想起和父亲的关系
写法核心：个人困惑铺垫 → 书的核心观点 → 观点照亮自己的经历 → 延伸到普遍困境 → 自己的修正或反对
小标题：禁止。本质是散文式思考流。

**概念解剖型**
拿一个心理学/哲学概念讲透，并落到生活。不是科普，是"这个词藏着我没说清楚的东西"。
- 认知失调 → 从自己买贵东西后的心理活动切入
- 防御机制 → 从自己拖延的场景切入
写法核心：概念引入（从一个生活场景，不是教科书式）→ 准确表述 → 生活化举例 → 反例和边界 → 个人体悟
小标题：允许。解剖多个概念或多个层面时，小标题能帮读者建立认知地图。

**现象追问型**
观察到一个日常现象，反复追问为什么。不是观察报告，是"越想越有意思"。
- 大家都在刷手机但都说无聊
- 年轻人开始集体怀旧
- 朋友圈越来越安静
写法核心：现象描述 → 第一层解释（表面）→ 追问 → 第二层（机制）→ 再追问 → 第三层（结构/哲学）
小标题：允许。当追问的层次较多或需要拆解多个相关现象时，小标题能标示推进的节奏。

**方法论型（自我实践）**
把自己长期实践的一个方法/原则系统分享。不是教程，是"我把压箱底的东西掏给你"。
- 我是怎么写日记三年的
- 我戒掉短视频的30天
- 我每天清晨的一小时
写法核心：谦逊铺垫 → 具体做法（可执行）→ 真实改变 → 学习成本和失败点 → 适用边界
小标题：允许。方法论天然适合分点骨架。

## 3. 推进结构详解

### 为什么需要推进结构

深度文章的核心不是"讲了什么观点"，是"怎么一步步走到这个观点的"。读者读完应该感觉"我跟着作者想了一圈"，不是"作者告诉我了一个结论"。

如果结论来得太容易，整个推进就垮了。如果停在第一层就草草收束，文章就浅了。

### 三段式追问

最基本也最常用的推进结构。

**模板：**
```
现象描述
 ↓
第一层解释（多数人怎么理解的）
 ↓
"但真的是这样吗？" / "但事情真的这么简单吗？"
 ↓
第二层解释（机制/结构层面）
 （此处可引入锚点）
 ↓
"再往深想一层呢？" / "再退一步讲呢？"
 ↓
核心洞察（存在层/哲学层）
```

**关键要求：**
- 每一层都必须比上一层更接近结构性问题
- 两次追问之间必须有实质深化，不能是同一层换个说法
- 第一次追问后可以偏离主线讲一个案例或锚点，但必须用扣主线句拉回

### 反向论证

在揭示核心观点前，先满足读者的期待，然后温和地打破它。

**模板：**
```
你以为是A（多数人的常识）
 ↓
其实一开始我也以为是A（卸掉"我要否定你"的姿态）
 ↓
但后来我发现……（温和的转折）
 ↓
其实是B（你的核心观点）
```

**关键要求：**
- 姿态永远是"我也曾经这样想"，不是"你们都错了"
- 转折要温和，不是颠覆
- 转折后必须有具体论证支撑，不能只是"不对，应该是B"

### 层层剥开

不直接讲结论，每一句在上一句基础上再推进一层。

**模板：**
```
第一句：我们以为的X，其实只是Y
 ↓
第二句：而Y本身，也只是Z的一小部分
 ↓
第三句：Z的整体结构，其实是……
 ↓
第四句：而Z背后，还藏着更深的……
```

**关键要求：**
- 每一句都要在上一句的基础上往前推一步，不是平行陈述
- 读者能跟着你的推理走，不是被动接收
- 不要跳步，每一步的过渡要让读者"哦，原来是这样"

### 对反方的承认

讲观点前先完全站在反方立场把反方讲透。

**模板：**
```
我非常理解这种说法（承认姿态）
 ↓
你可能也会觉得……（具体复述对方的处境和论证）
 ↓
我自己很长一段时间也是这么想的（卸掉对抗感）
 ↓
但后来我发现……（切入自己的不同视角）
 ↓
这里藏着一个很微妙的问题……（指出对方论证里的盲点）
```

**关键要求：**
- 必须具体地复述对方，不能只是"我理解你的观点"
- 承认对方合理的地方，不是假装承认
- 切入自己的观点时，要指出对方论证里的具体盲点，不是直接否定

### 拒绝一步到位

深度文章的核心特征。

**反面例子：**
> 所以说，深度思考很重要，我们都要培养深度思考的习惯。

这种结论来得太容易，整个推进就垮了。

**应该改成：**
> 写到这里，我大概可以这么讲。
> 深度思考不是一种能力，是一种选择。
> 是在每一次可以选择刷手机还是看书的时候，选择看书。
> 在每一次可以选择快餐内容还是慢内容的时候，选择慢的。
> 这种选择不是一次性的事，是每天每时每刻的事。

让结论有"走到这一步是水到渠成"的感觉，不是"所以我告诉你"。

### 横向铺陈结构

推进不一定是纵向的，也可以是横向的——并列展开多个相关概念。

**适用场景**：
- 盘点型文章（5个底层逻辑、3条原则、4种思维误区）
- 方法论型文章（3层结构、4个阶段、5个步骤）
- 现象追问型文章（拆解多个相关现象的共同根因）

**模板：**
```
总起（抛出要盘点的母题）
 ↓
点1：是什么 + 为什么 + 怎么做
 ↓
点2：是什么 + 为什么 + 怎么做（与点1有递进或对照关系）
 ↓
点3：是什么 + 为什么 + 怎么做（与前两点有递进或对照关系）
 ↓
...
 ↓
总结（不是简单重复，是点出这些点之间共同指向的母题）
```

**关键要求：**

- 每个点内部仍然要有"是什么+为什么+怎么做"的纵向推进，不能只是一个判断加几个例子。
- 点与点之间要有内在逻辑——递进（从浅到深、从因到果、从现象到本质）或对照（正反、内外、古今）。完全平行的罗列会让文章变成清单，失去深度。
- 点的数量建议3-5个。少于3个说明结构感不够，多于5个说明在硬凑分点。
- 核心洞察应该藏在点的排列顺序和点之间的呼应里，不是靠某一个点单独撑起。

**适合配小标题**：横向铺陈的文章天然适合用小标题分点，但小标题本身要金句式（不是描述性标签），具体见 SKILL.md 的"小标题使用规范"。

## 4. 锚点工作法

### 锚点的本质

锚点不是装饰，是照亮。一个好的锚点，能把文章里原本说不清楚的感受，用一个外部的参照物讲清楚。

如果一个锚点拿掉之后文章也成立，说明这个锚点是硬凑的。
如果一个锚点需要你花很大力气去解释，说明这个锚点不合适。
如果一个锚点你只是道听途说式地知道，不要用——会暴露装深刻。

### 锚点的类型

**哲学锚点**：存在主义、斯多葛、虚无主义、禅宗
- 讲意义感缺失时连到加缪的西西弗斯
- 讲焦虑时连到海德格尔的"向死而生"
- 讲控制欲时连到斯多葛的爱比克泰德
- 讲荒诞感时连到加缪的《局外人》

**心理学锚点**：认知失调、防御机制、习得性无助、心流、依恋理论
- 讲拖延时连到弗洛伊德的防御机制
- 讲专注时连到契克森米哈伊的心流
- 讲无力感时连到塞利格曼的习得性无助
- 讲自我矛盾时连到费斯廷格的认知失调

**文学锚点**：卡夫卡、村上春树、陀思妥耶夫斯基、鲁迅、加缪
- 讲现代人异化时连到卡夫卡的《变形记》
- 讲孤独感时连到村上春树的某种意象
- 讲国民性时连到鲁迅

**历史锚点**：某个历史时刻或人物
- 讲信息过载时连到古登堡印刷术发明后的焦虑
- 讲当代倦怠时连到工业革命早期工人对钟表的反抗
- 讲注意力经济时连到电视普及后的"娱乐至死"讨论

**科学锚点**：熵增、混沌、神经可塑性
- 慎用，必须真的准确理解才能用
- 容易暴露装深刻（用得不准）或装理工（用得不必要）

### 锚点使用原则

**一篇文章一个主锚点就够。**
不要堆砌。堆砌反而暴露装深刻。如果文章里出现了三个以上的锚点，大概率是硬凑的。

**锚点必须照亮主题，不是装饰。**
测试方法：拿掉这个锚点之后，文章是否缺一块？如果不缺，说明锚点是硬凑的。

**准确表述锚点。**
不要道听途说式转述。"韩炳哲说……"和"韩炳哲在那本书里反复强调的是……"完全不同。如果不确定锚点的准确表述，让AI查证后再用。

**锚点要有个人化的接触方式。**
什么时候第一次读到、当时什么感觉、反复读了多少遍。让锚点变成"我的"而不是"百度百科的"。

好的引入方式：
> 后来我读到韩炳哲那本《倦怠社会》，突然感觉之前所有的困惑都被一个词照亮了。

坏的引入方式：
> 韩裔德国哲学家韩炳哲在《倦怠社会》一书中提出了"功绩社会"的概念...

### 锚点出现的时机

**最好的位置是第二层解释处。**
- 第一层是常识性解释，不需要锚点
- 第二层是机制/结构层，锚点刚好能起到"照亮"的作用
- 核心洞察层是作者自己的话，不需要锚点（否则会显得引用别人代替自己说话）

**不要在开头用锚点。**
开头应该是具体的场景或困惑，不是哲学概念。

**不要在结尾用过多锚点。**
结尾是作者自己的洞察或回环呼应，引用收尾可以用一个锚点，但不能堆砌。

### 锚点组合的禁忌

- 不要一篇文章里同时用加缪和海德格尔（同是存在主义，显得掉书袋）
- 不要用自己不熟悉的锚点（被读者一问就露馅）
- 不要用太大众的锚点（"马斯洛需求层次"已经被用滥了，除非有新角度）
- 不要用太小众的锚点（读者看不懂，反而疏离）

## 5. 内容节奏

### 节奏的本质

节奏不是文笔问题，是一套让读者一路往下读的推进系统。

好的节奏像呼吸：吸气（推进）→ 呼气（停顿）→ 吸气（再推进）。读者跟着你的节奏走，不会累，也不会走神。

深度文章的节奏比一般文章慢一点——因为读者需要时间消化每一层的推进。但慢不等于拖，每一段都要往前走。

### 扣主线句

偏离主线之后必须高频出现扣主线句，不需要长，一句就够，但频率要高。

好的扣主线句：
- "说远了，拉回来。"
- "回到刚才那个问题。"
- "我这儿扯得有点远，但其实是想说……"
- "顺着这个再聊一点。"
- "这事儿其实还是在讲……"

频率：每偏离主线一次，必须有扣主线句拉回。一篇3000-6000字的文章，扣主线句至少5-8次。

### 留白

一句话独立成段的留白效果。深度文章的留白不是制造"卧槽"的冲击，是让读者"停下来想一秒"。

使用频率：全文3-6次。过度使用反而显轻浮。

使用时机：
- 情绪高点
- 推进的转折点
- 核心洞察出现之后
- 结尾的哲思余韵

### 节奏优先级

深度（底子）→ 推进（方向）→ 主线（绳子）→ 节奏（牵引力）

深度是底子，没有深度再好的节奏也撑不起来。
推进是方向，没有推进的文章是原地打转。
主线是绳子，所有段落都挂在主线上。
节奏是牵引力，让读者跟着你走。

### 最忌讳的节奏问题

**突然偏离主线很远再硬拽回来。**
读者要花脑力"顺逻辑"，心流立刻断掉。

**连续多段句式长度相近。**
读起来像念经，节奏呆板。

**没有留白的长段落。**
读者读到一半就累了，需要休息点。

**推进太快。**
结论来得太容易，读者会觉得"这就完了？"。

**推进太慢。**
在同一层绕来绕去，读者会觉得"你怎么还没讲清楚"。

### 对标参考

深度思考类文章的对标不是AI科技博主，而是：
- 写散文的作家（村上春树的随笔、梁文道的小篇幅）
- 写思想札记的学者（刘瑜、刀尔登）
- 写公共议题的深度作者（兔撕鸡、孟常）
- 写心理成长的咨询师（武志红的部分文章）

读这些人的文章，感受他们的节奏、推进、锚点使用方式。不是为了模仿，是为了校准自己的语感。
